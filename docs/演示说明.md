# 演示说明

## 演示目标

输入 GitHub PR 链接后，系统自动获取 PR 变更，生成包含变更总结、风险代码、Review 建议和测试建议的 AI Review 报告。

## 演示前准备

1. 安装依赖：

   ```bash
   npm install
   ```

2. 复制环境变量示例：

   ```bash
   cp backend/.env.example backend/.env
   cp frontend/.env.example frontend/.env
   ```

3. 在 `backend/.env` 中配置：

   ```env
   DEEPSEEK_API_KEY=your_deepseek_api_key
   DEEPSEEK_BASE_URL=https://api.deepseek.com
   DEEPSEEK_MODEL=deepseek-v4-flash
   ```

   如需提高 GitHub API 限额，可额外配置 `GITHUB_TOKEN`。

4. 启动项目：

   ```bash
   npm run dev
   ```

## 演示流程

1. 打开前端页面 `http://localhost:5173`。
2. 展示页面标题“AI PR Review 助手”和 PR 链接输入框。
3. 输入一个公开 GitHub PR 链接。
4. 点击“开始分析”。
5. 展示 loading 状态，说明系统正在获取 PR 信息、整理 diff 上下文并调用模型。
6. 展示 Review 报告：
   - PR 变更总结。
   - 风险等级。
   - 风险代码列表。
   - Review 建议。
   - 测试建议。
   - Markdown 报告。
7. 点击“复制 Markdown 报告”，展示“复制成功”提示。
8. 将 Markdown 报告粘贴到文本编辑器或 PR 评论框，展示可用性。

## 讲解要点

- 项目通过 GitHub REST API 获取真实 PR 信息，不依赖手动复制 diff。
- 系统对超长 patch 做截断，避免 prompt 过长。
- 模型输出固定结构，前端可稳定展示。
- 当模型返回非 JSON 时，后端有兜底处理。
- API Key 均通过环境变量配置，不提交到仓库。
- 开发过程按 PR 拆分，每个 PR 只完成一项独立功能。

## 异常演示

可准备以下异常场景说明系统稳定性：

- 输入空链接：提示 PR URL 不能为空。
- 输入非 GitHub 链接：提示必须为 GitHub PR 链接。
- 输入不存在的 PR：提示 PR 不存在。
- DeepSeek Key 未配置：提示需要配置 `DEEPSEEK_API_KEY`。

## 演示结论

本项目展示了从 PR 链接到 AI Review 报告的完整闭环，覆盖题目要求中的变更总结、风险识别、Review 建议、模型选择、上下文获取、误报漏报控制、响应速度优化、使用体验和未来扩展方向。
